## Phase 1 — 像素編輯器：繪圖核心

**目標**
讓玩家能在 16x16 畫布上繪製像素、選色、並即時預覽結果。

**背景與動機**
自定義像素編輯器是整個產品的核心差異化功能。Web Demo 的主要目的是讓使用者體驗「創作寵物」，因此編輯器的基礎繪圖能力必須優先完成。目前已有像素渲染基礎（`usePixelCanvas`），可在此之上擴展。

**功能範疇**

| 功能 | 優先級 | 說明 |
|------|--------|------|
| 16x16 互動畫布 | Must | 滑鼠點擊／拖曳可塗色的格子畫布 |
| 顏色選擇器 | Must | 提供調色盤供玩家選取當前繪製顏色 |
| 橡皮擦工具 | Must | 將格子設為透明（清除顏色） |
| 即時預覽 | Must | 編輯同時旁邊顯示實際渲染效果（動畫播放） |
| 清除全畫布 | Should | 一鍵清空目前幀的所有像素 |
| 填色工具 | Could | Flood fill，點擊後填滿同色連通區域 |

**用戶故事**
- 身為一個創作者，我想要在 16x16 格子畫布上用滑鼠塗色，以便設計我的寵物外觀。
- 身為一個創作者，我想要從調色盤選擇顏色，以便精確控制像素色彩。
- 身為一個創作者，我想要即時看到渲染後的動畫效果，以便確認成品看起來符合預期。

**驗收標準**
- [ ] 畫布顯示 16x16 格子，每格可獨立點擊塗色
- [ ] 拖曳滑鼠可連續塗色（不需逐格點擊）
- [ ] 調色盤可選色，選色後繪製即套用新顏色
- [ ] 橡皮擦可清除單格顏色（設為透明）
- [ ] 旁邊即時預覽區域以實際像素動畫呈現目前狀態
- [ ] 清除按鈕可清空整個畫布

**範疇外**
- 動畫幀管理（多幀編輯）— Phase 2 處理
- 存檔 / 讀檔 — Phase 2 處理
- 圖層系統
- 匯出圖片（PNG）
- 複製 / 貼上

**PR 拆分建議**
1. `PR-1-1`：建立 `/editor` 路由與頁面骨架（空白頁面、導覽連結）
2. `PR-1-2`：實作 16x16 互動畫布元件（點擊塗色、拖曳連續繪製）
3. `PR-1-3`：實作顏色選擇器元件，與畫布串接
4. `PR-1-4`：實作橡皮擦工具與清除全畫布按鈕
5. `PR-1-5`：實作即時預覽區塊（重用 `usePixelCanvas`，播放動畫）

## Phase 2 — 像素編輯器：動畫幀管理與存檔

**目標**
讓玩家能管理多個動畫幀、定義動作，並將創作成果儲存在本地。

**背景與動機**
單幀靜態圖不足以呈現寵物的生命力。動畫是讓玩家「愛上自己創作」的關鍵。存檔讓創作成果不會因重新整理頁面而消失，是基本的使用者體驗保障。

**功能範疇**

| 功能 | 優先級 | 說明 |
|------|--------|------|
| 幀列表面板 | Must | 顯示目前動作的所有幀，可新增、刪除、切換 |
| 幀複製 | Must | 複製現有幀作為新幀的起點，方便逐幀修改 |
| 動作管理 | Must | 支援多個動作（如：idle、walk、sleep），可新增與切換 |
| 動畫播放速度控制 | Should | 提供 FPS 調整滑桿或輸入框 |
| 存入 localStorage | Must | 自動或手動儲存所有幀與動作資料至 localStorage |
| 從 localStorage 讀取 | Must | 頁面載入時自動還原上次創作狀態 |
| 匯出 JSON | Could | 將整份寵物資料下載為 `.json` 檔案 |
| 匯入 JSON | Could | 讀取 `.json` 檔案還原創作 |

**用戶故事**
- 身為一個創作者，我想要新增多個幀並逐幀編輯，以便製作流暢的動畫。
- 身為一個創作者，我想要定義不同動作（如走路、休息），以便讓寵物有多種表現。
- 身為一個創作者，我想要關閉頁面後再回來仍保有我的創作，以便不用每次重做。

**驗收標準**
- [ ] 幀面板顯示當前動作所有幀的縮圖
- [ ] 可新增空白幀與複製現有幀
- [ ] 可刪除幀（至少保留 1 幀）
- [ ] 可新增動作並自訂名稱，切換動作時畫布顯示對應幀
- [ ] 動畫播放速度可調整
- [ ] 離開頁面再回來，創作內容完整還原
- [ ] 匯出 JSON 下載完整寵物資料（Could，若時間允許）

**範疇外**
- 幀重新排序（拖曳換位）
- 復原 / 重做（Undo/Redo）
- 線上同步 / 雲端存檔
- 多隻寵物管理

**PR 拆分建議**
1. `PR-2-1`：實作幀列表面板（顯示縮圖、切換幀）
2. `PR-2-2`：實作新增幀、複製幀、刪除幀功能
3. `PR-2-3`：實作動作（action）管理——新增、切換、命名
4. `PR-2-4`：實作 localStorage 存讀（自動儲存 + 頁面載入還原）
5. `PR-2-5`：實作動畫播放速度控制
6. `PR-2-6`：實作匯出 / 匯入 JSON（若 Phase 2 時間允許）

## Phase 3 — 桌面版（Electron）整合

**目標**
將 Web 版包裝為 Electron 桌面應用，實現透明置頂視窗與滑鼠穿透功能。

**背景與動機**
桌面放置是核心產品體驗的基礎。透明置頂讓寵物真正「活在桌面上」，是與一般網頁寵物最大的差異。此階段驗證技術可行性後，才能推進挖礦等遊戲邏輯。

**功能範疇**

| 功能 | 優先級 | 說明 |
|------|--------|------|
| Electron 基礎整合 | Must | Nuxt 4 + Electron 開發環境建立 |
| 透明背景視窗 | Must | 應用程式視窗背景為透明 |
| 置頂切換 | Must | 可切換寵物是否顯示在所有視窗之上 |
| 滑鼠穿透 | Must | 平時不攔截滑鼠事件，只在需要互動時接收 |
| 寵物在螢幕底部走動 | Must | 載入玩家創作的寵物並在螢幕底部自動左右移動 |

**用戶故事**
- 身為一個玩家，我想要寵物透明地顯示在桌面上，以便不影響我正常使用電腦。
- 身為一個玩家，我想要切換置頂開關，以便決定寵物是否遮蓋其他視窗。

**驗收標準**
- [ ] Electron 應用可在本地建置並啟動
- [ ] 視窗背景透明，寵物像素正常顯示
- [ ] 置頂切換按鈕功能正常
- [ ] 平時滑鼠可穿透寵物區域，不影響底層應用操作
- [ ] 寵物在螢幕底部（工作列上方）自動左右走動並播放行走動畫

**範疇外**
- 挖礦邏輯
- 道具系統
- 自動更新機制

**PR 拆分建議**
1. `PR-3-1`：建立 Nuxt 4 + Electron 開發環境（main process 設定）
2. `PR-3-2`：實作透明背景視窗與置頂切換
3. `PR-3-3`：實作滑鼠穿透（預設穿透，互動時暫停穿透）
4. `PR-3-4`：實作寵物在螢幕底部自動左右走動

## Phase 4 — 掛機挖礦與道具系統

**目標**
實現全自動挖礦掉落邏輯、驚喜動畫回饋，以及基礎道具圖鑑。

**背景與動機**
挖礦是讓玩家「掛著遊戲」的驅動力，也是 Steam 版的核心留存機制。稀有道具的驚喜感是社群分享的引爆點。

**功能範疇**

| 功能 | 優先級 | 說明 |
|------|--------|------|
| 隨機掉落算法 | Must | 寵物走動期間隨機觸發道具掉落 |
| 道具定義資料 | Must | 定義初版道具清單（名稱、稀有度、圖示） |
| 驚喜動畫與標示 | Must | 挖到道具時出現「！」並播放驚喜動畫 |
| 點擊領取互動 | Must | 玩家點擊寵物後收獲道具，驚喜標示消失 |
| 道具圖鑑 UI | Must | 顯示所有道具格子，已收集者亮起，未收集者灰顯 |
| 道具持久化儲存 | Must | 收集到的道具儲存至本地，重開不消失 |
| 截圖功能 | Could | 內建截圖，方便分享稀有道具 |

**用戶故事**
- 身為一個玩家，我想要讓寵物自動幫我挖礦，以便在工作時也能被動收集道具。
- 身為一個玩家，我想要看到驚喜動畫提示，以便知道什麼時候有道具可以領取。
- 身為一個收藏家，我想要看到完整圖鑑，以便追蹤還缺哪些道具。

**驗收標準**
- [ ] 寵物走動時依設定機率自動觸發掉落
- [ ] 挖到道具時出現「！」標示並播放對應動畫
- [ ] 玩家點擊後道具計入圖鑑，標示消失
- [ ] 圖鑑介面正確顯示已收集與未收集狀態
- [ ] 關閉並重開應用，收集紀錄完整保留

**範疇外**
- 功能性道具（改變數值或畫布規格）
- 道具交易或玩家間互送
- 線上排行榜

**PR 拆分建議**
1. `PR-4-1`：定義道具資料結構與初版道具清單
2. `PR-4-2`：實作隨機掉落計時器與觸發邏輯
3. `PR-4-3`：實作驚喜動畫幀與「！」標示顯示
4. `PR-4-4`：實作點擊領取互動（滑鼠穿透暫停 → 領取 → 恢復）
5. `PR-4-5`：實作道具圖鑑 UI
6. `PR-4-6`：實作道具收集資料持久化儲存

## Phase 5 — Web Demo Landing Page 優化與 Steam 導流

**目標**
優化 Web Demo 頁面，讓訪客體驗像素編輯器後被引導至 Steam 下載完整版。

**背景與動機**
Web Demo 是最主要的獲客管道。Landing Page 的轉換率直接影響 Steam 銷量。此階段在產品功能完整後進行，確保展示的是最佳狀態。

**功能範疇**

| 功能 | 優先級 | 說明 |
|------|--------|------|
| 編輯器入口優化 | Must | 首頁明確引導訪客進入像素編輯器體驗 |
| Steam 下載 CTA | Must | 明確的 Steam 頁面連結按鈕 |
| 社群截圖分享 | Should | 編輯器內建分享功能，方便玩家曬創作 |
| SEO 基礎設定 | Should | meta title、description、OG 圖片 |
| 效能優化 | Should | 首屏載入速度、動畫效能 |

**用戶故事**
- 身為一個訪客，我想要快速試玩像素編輯器，以便判斷這款遊戲是否符合我的喜好。
- 身為一個玩家，我想要一鍵分享我的寵物創作，以便讓朋友看到並引發興趣。

**驗收標準**
- [ ] 首頁有明確的「開始創作」入口，一步進入編輯器
- [ ] 編輯器頁面有 Steam 下載按鈕（連結可配置）
- [ ] 分享功能可截圖編輯器畫面（含寵物動畫）
- [ ] Lighthouse 效能分數 ≥ 80

**範疇外**
- 後端用戶系統
- 作品社群展示牆
- A/B 測試框架

**PR 拆分建議**
1. `PR-5-1`：首頁 CTA 重新設計，加入「開始創作」入口
2. `PR-5-2`：Steam 下載按鈕元件（支援連結配置）
3. `PR-5-3`：編輯器截圖分享功能（Canvas 轉圖片）
4. `PR-5-4`：SEO meta 設定與 OG 圖片
