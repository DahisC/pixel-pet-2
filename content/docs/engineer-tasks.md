**實作順序說明**

任務 1 → 2 → 3，所有任務在 `phase-1` 分支上進行，每個任務對應一個獨立 commit。

- **任務 1** 是核心，路由、store、畫布互動一起到位
- **任務 2** 依賴任務 1 的 store 結構，擴充工具與選色
- **任務 3** 需求變更，改由 Phase 2 納入即時預覽

**PR 流程**

工程師的 PR 目標分支為 `dev`（不是 `main`）。`main` 由使用者從 `dev` 手動合併。流程如下：

1. 工程師在功能分支完成實作後，開 PR 合併至 `dev`
2. 使用者審核並合併 PR 至 `dev`
3. 使用者確認 `dev` 穩定後，自行將 `dev` 合併至 `main`
