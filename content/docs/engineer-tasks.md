**PR 流程**

工程師的 PR 目標分支為 `dev`（不是 `main`）。`main` 由使用者從 `dev` 手動合併。流程如下：

1. 工程師在功能分支完成實作後，開 PR 合併至 `dev`
2. 使用者審核並合併 PR 至 `dev`
3. 使用者確認 `dev` 穩定後，自行將 `dev` 合併至 `main`

## Phase 2 — 像素編輯器：動畫幀管理與存檔

**分支：** `phase-2`

**進度總覽**
- 總任務數：5
- 已完成：0
- 進行中：0
- 待開始：5

**任務清單**

| # | 任務 | 狀態 | 備註 |
|---|------|------|------|
| 1 | Pinia store + 幀列表面板 | ⬜ 待開始 | `useEditorStore`、多幀資料結構、幀縮圖顯示、切換幀 |
| 2 | 幀操作：新增、複製、刪除 | ⬜ 待開始 | 依賴任務 1 的 store |
| 3 | 動作管理 | ⬜ 待開始 | 多動作（idle/walk/sleep）、新增、切換、命名 |
| 4 | 即時預覽 + 動畫播放速度 | ⬜ 待開始 | 重用 `usePixelCanvas`，FPS 控制，與 store 同步 |
| 5 | localStorage 存讀 | ⬜ 待開始 | 自動儲存，頁面載入時還原 |

**實作順序說明**

任務 1 → 2 → 3 → 4 → 5，所有任務在 `phase-2` 分支上進行，每個任務對應一個獨立 commit，PR 目標分支為 `develop`。

- **任務 1** 建立 store 骨架，是後續所有任務的基礎
- **任務 2** 依賴 store 的幀陣列結構
- **任務 3** 依賴 store 的動作切換邏輯
- **任務 4** 依賴完整的多幀 store 資料
- **任務 5** 在功能齊全後才做持久化，避免 store 結構未定就寫死格式

**技術筆記**

- `useEditorStore`（Pinia）需管理：動作列表、當前動作、當前幀 index、像素陣列、當前顏色、當前工具
- 幀縮圖使用 `usePixelCanvas` 以小 scale（如 2px）渲染
- localStorage key 建議：`pixel-pet-editor`，存整份動作+幀資料的 JSON
